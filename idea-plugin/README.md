# AI File Cleaner · AI 文件清理 (IntelliJ IDEA 插件)

> 识别工作区中由 **AI 生成的无用文件 / 临时文件 / 项目配置 / AI 配置 / 可疑文件**，并一键清理，**保证本地干净**。

适用于使用 AI 编程助手后产生大量临时/草稿/配置残留的场景：导入或新建文件后自动识别，及时删除无用文件、把配置类文件加入忽略/排除、对不可预测的可疑文件支持**一键转存**到指定目录或**一键删除**。

---

## ✨ 功能

| 文件类型 | 默认处理 | 说明 |
| --- | --- | --- |
| 🤖 **AI 生成的无用文件** | 删除 | 通过文件名规则（`scratch_*`、`*.generated.*`、`demo_*` 等）与内容标记（`@generated`、`Generated by AI`、`DO NOT EDIT` 等）识别 |
| 🧹 **临时文件** | 删除 | `*.tmp`、`*.bak`、`*~`、`.DS_Store`、`*.log`、缓存/编辑器残留等 |
| ⚙️ **项目配置文件** | 加入忽略/排除 | `.editorconfig`、`.eslintrc*`、`.prettierrc*` 等 |
| 🧠 **AI 配置文件** | 加入忽略/排除 | `.cursor/`、`.cursorrules`、`CLAUDE.md`、`AGENTS.md`、`.aider*`、`.continue/`、`.windsurf*` 等 |
| ❓ **可疑/不可预测文件** | 人工决策 | 未知二进制、随机文件名等 → **一键转存隔离** 或 **一键删除** |
| ✅ **正常文件** | 保留 | 源码与资源 |

- **配置 AI 接口**（OpenAI 兼容）：`Base URL` / `API Key` / `Model`，在本地规则无法判定时由 AI 二次判定。
- **导入/新建文件监听**：实时检测并给出可操作的通知（或开启“自动清理”静默处理；**可疑文件永远不会被自动删除**）。
- **加入忽略/排除** = 写入项目根 `.gitignore` + 目录在 IDE 中标记为 `excluded`（不破坏文件、不破坏构建，仅保持工作区/git 干净）。
- **删除安全网**：可选“删除前先转存到隔离目录”，避免误删。

## 🚀 使用

1. 用 IntelliJ IDEA（2023.3+）打开本目录（`idea-plugin/`）。
2. 运行 `./gradlew runIde` 启动带插件的沙箱 IDE，或 `./gradlew buildPlugin` 生成 `build/distributions/ai-file-cleaner-*.zip`，通过 **Settings → Plugins → ⚙️ → Install Plugin from Disk…** 安装。
3. 打开 **Settings → Tools → AI 文件清理**，填写 `Base URL / API Key / Model`，点「测试连接」。
4. 触发清理：
   - 菜单 **Tools → AI 文件清理：扫描…**，或在 **项目视图右键 → AI 文件清理** 子菜单。
   - 扫描结果对话框中勾选文件、调整每行操作（删除 / 加入忽略·排除 / 转存隔离 / 保留），点「执行所选操作」。
   - 对单个文件可直接右键「转存隔离 / 删除 / 加入忽略·排除」。

### 隔离目录

转存默认到 `<项目根>/.ai-cleaner-quarantine/<时间戳>/<原相对路径>`，可在设置中自定义为任意指定目录。

## ⚙️ 可配置规则

设置页中所有规则均可自定义（glob，逗号或换行分隔，大小写不敏感）：

- 临时文件规则
- AI 配置文件规则
- 项目配置文件规则
- AI 生成无用文件（文件名）规则
- AI 生成标记（内容关键字，每行一个）

## 🏗️ 技术栈

- Kotlin + IntelliJ Platform Gradle Plugin 2.x（目标 IDE：IntelliJ IDEA Community 2024.2）
- API Key 通过 IDE `PasswordSafe` 安全存储；其余配置持久化到应用级设置。
- AI 调用使用 JDK 内置 `HttpClient` + 平台自带 Gson，无额外运行时依赖。

## 🔒 安全说明

- API Key 仅保存在本机凭据库（PasswordSafe），不写入项目文件。
- 默认仅给出建议/通知，**不会自动删除**任何文件，除非你显式开启“自动清理”（即便如此，可疑文件也只会提示、绝不自动删除）。
- 推荐开启“删除前先转存到隔离目录”，删除即可恢复。
